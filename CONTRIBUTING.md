# Contributing

Thanks for wanting to contribute to HeyYou!

## PRs

This project uses trunk-based development with small, focused PRs. Each PR
should be a single logical change.

1. Fork the repo and create your branch from `main`.
2. Write tests for any new functionality.
3. Ensure all tests pass: `make test`
4. Open a PR with a clear description of what it does and why.

## Code style

- No comments unless the code genuinely cannot be made self-explanatory.
- Follow existing patterns in the codebase.
- Prefer clarity over brevity.

## Commit messages

Write commit messages in the imperative: "Add monitoring engine" not "Added monitoring engine".
